Advanced: Pull Requests, Rebasing and Guidelines: Contributing to Open Source

Git is great to know for software development generally, but it's also used by many open-source projects. An essential part of the learning how to code is working on actual projects and codebases and a great way to do that is contributing to open-source projects. Before you jump in feet first into a project and start hacking away it's really important to know some basic guidelines for contributing to open-source.

Pull Requests

We mentioned pull requests briefly in the first section of this module, but to recap: A pull request is when a contributor asks to merge a branch of development with another branch. This involves the maintainers of the second branch to review and comment on the contributor's code. Sometimes, a pull request is approved and the contributor's branch is merged into the other branch. A pull request typically contains the files to be changed if the two branches were to merge, description of what those changes are / accomplish, and a forum of some sort for the different parties to discuss the pull request. Remember, a pull request is not a part of the core Git software, but rather is a feature of Git services like Github or Gitlab.

Pull requests are a big part of collaborating on Github and other Git services.

Rebasing

In the process of a pull request, you might realize something in your branch is wrong. The process of undoing, fixing or removing commits is called rebasing. It's best practice to change a Git project by another commit, rather than trying to rework a previous commit. However, sometimes it can't be avoided. Here are some resources to help in those times:

  • Undoing, Fixing, or Removing Commits in Git A "Git Choose-Your-Own-Adventure," this is a nice, simple site that asks questions about your rebasing situation and recommends a solution.
  • Dang it, Git! A less profane version of a popular Git troubleshooting guide.
  • How to Undo (Almost) Anything with Git Lays out a series of scenarios and suggests ways of undoing the error.
