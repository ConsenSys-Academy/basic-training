Setting Up Github

In this section, we're going to setup a Github account and make sure we can push to it from our command line. Even though there are other Git websites, Github is the most popular. We are not saying it's the best, but it is very common with multi-lingual, extensive support. Another option, if you prefer, is Gitlab, but if you're doing the Bootcamp, we'd appreciate it if you used Github.

You can use $ git clone from your command line without a Github account or Personal Access Token (see below). You can also write commits, create pull requests, and create repositories from the Github website in your browser. But that's not always the fastest way to work and you should setup Git to interact with Github from your command line. Follow the steps below to do so.

Sign Up for a Github Account

Click here to be taken to the sign-up page for Github. Pick a username, provide your e-mail address and create a password. Once you've registered, Github requires you to confirm your e-mail address. After that, you should be ready to go!

Setup Github on Command Line

Install Git

Git comes installed default on almost all Mac and Linux systems. Make sure you have Git 1.7.10 or later installed by running $ git --version on your command line. If you get an error or have an earlier version, follow the instructions for installing Git on Mac or Linux here. If you're using Windows, be sure to install WSL and make sure Git is installed on your WSL.

Personal Access Token

Next, you need to generate a Personal Access Token from Github. To generate this token, be sure you are registered on Github and follow these instructions. The Personal Access Token is like your password and should not be published or shared.

Git Credential Caching

When you're pushing a commit to a Github remote repository, Github will require you to provide your username and Personal Access Token from the command line. You can provide this every time or, if you use Credential Caching, you can do it once and Git will remember it until you change it.

To use Git Credential Caching, you'll need Git version 1.7.10 or higher and a Github username and Personal Access Token. Once you have those, you can follow the instructions here on how to do Credential caching for both Mac and Linux.

While it's a bit tricky to get setup, it will save you a lot of time in the long run. If you're having a lot of trouble with it, you don't have to struggle on. It's not absolutely necessary, but it is nice to have.
