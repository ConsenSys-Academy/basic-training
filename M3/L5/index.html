Shortcuts, Remapping, Customization and Aliasing

Now that we've got the basics covered, let's start to make things a bit more fun and interesting. Customization can go a long way to making you feel more comfortable and at home on the command line, or at least not quite as dreadful. We'll start by going over some keyboard shortcuts that will improve your quality of life, then talk about remapping your keyboard. Last, we'll give a few options for customizing your terminal emulator.

Shortcuts

Here are some common keyboard shortcuts for the command line:

  • CTRL + c Interrupts and quits the process currently running. Crucial in stopping terminal line programs
  • CTRL + e Moves your cursor to the beginning of the line
  • CTRL + a Moves your cursor to the end of the line
  • CTRL + l Clears your screen without typing clear
  • CTRL + k Delete from current cursor position to end of the line
  • CTRL + t (Linux) / Escape + t (Mac) Will swap the two words that appear immediately before the cursor
  • CTRL + r Search your previous command history, easier than using up or down arrows

For more Linux shortcuts, please see this article. For more Mac shortcuts, please see this article.

Remapping

Remapping is the process of changing one key on your keyboard to be another. For example, you can see in the list above, many of these shortcuts use CTRL. Because of this, a common remapping developers use is making your computer think your "Caps Lock" key is actually CTRL. You can find out how to do that here. You use your CTRL key way more often than you do your Caps Lock. How often do you actually use your Caps Lock?

As you continue to use your terminal, you may find other keys you'd like better access to. Consider remapping them if it makes your life a bit easier.

Remapping can also be done with shortcuts within your terminal. If you find yourself using a shortcut but strain a bit to hit the keys, consider remapping that shortcut to something easier for your hands.

Customization: Color and Font

Many terminal clients allow you to customize their color, font, sizing, and layout. This not only is fun, but also can help with readability and eye strain. We'll look at a few options for colors and fonts

Colors

The best way to change the color of your terminal is to find a good color theme. Color themes are sets of colors with an overall coherence that can be loaded in all at once. As an added bonus, you can replicate a good color theme in both your terminal client and code editor.

Fonts

Changing fonts on your terminal can be very impactful for readability and strain. There are a lot of options (if you're doing a lot of Git, vim and Python work, you might want to consider Powerline, but it's quite advanced.). We'll link to this Wes Bos article which lists fonts good for your terminal. We wouldn't recommend purchasing a font if you're just starting, but play around with some free ones!

Another important aspect of font is size, kern spacing and line height spacing. Again, it's to your own preferences, so experiment and see what you like!

Aliases

Aliases are short commands that the terminal interprets as longer commands. When they're applied to commands you run often, they can save significant time. A very common alias people create is mc DIRECTORY_NAME which the terminal will interpret as mkdir DIRECTORY_NAME && cd DIRECTORY_NAME, essentially creating and navigating to the folder in one command.

Creating aliases can be tricky, so if you're still figuring out the terminal, feel free to come back to this lesson.

For those interested, we'd recommend a two step process for developing effective aliases:

  1. Find what commands you use most frequently. There are a few ways to do this, but, if your history is long enough, the following command may work on a UNIX-like system: history | awk '{print $2}' | sort | uniq -c | sort -rn | head. Note: to make this most effective, you typically must increase your history and specify terminal to save history when exiting, otherwise you will only have the commands you've used recently. Please see the next section on Shell Frameworks to see how to do this.

  2. Once you've found which commands you use the most, you need to locate the correct file to create your aliases. This is typically a profile or a run command (rc) file but which run command file depends on your shell. If you're using bash, you'll want to create or edit ~/.bash_profile or ~/.bashrc. If you're using zsh, the default for MacOS 10.15 Catalina and later, you'll want to edit ~/.zprofile or ~/.zshrc. See this Superuser post for more information on profile and rc files.

Once you've found the command(s) you'd like to create the aliases for and the proper place to put them, you can create an alias using the following syntax options (taken from this site):

      alias name=value
      alias name='command'
      alias name='command arg1 arg2'
      alias name='/path/to/script'
      alias name='/path/to/script.pl arg1'

Last, once you've saved your changes to the profile or rc file, you can activate those changes by running (from the command line) $ source ~./YOUR_PROFILE_OR_RC_FILE

To double check your alias has been added, type $ alias to list all available aliases in your terminal

Common Commands to Alias

Below are some common commands people for which make aliases. It should give you a sense of the goal of aliases and what they look like.

  • alias ..='cd ..' You can add on to this, for example alias ...='cd ../..'
  • From Sarah Drasner: alias gi='git add -A && git commit -m', which allows you to run gi 'your commit message here'
  • From Sarah Drasner: alias gr='grep --color -r', which searches recursively from your current folder and colorizes output, i.e. gr 'assets'

Hopefully, this has given you a few ideas on how to start aliasing and customizing in general. Other tips are just an internet search away!
