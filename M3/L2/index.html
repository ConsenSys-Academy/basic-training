What is the Command Line Interface (CLI)?

The command line interface (CLI) is the basic non-graphic interface for your computer. It is sometimes also called the terminal, the shell, or the bash client. In this section, we'll go over some basic concepts.

Everything is a File

An overarching philosophy with command line clients is the concept that Everything is a File. This means that every item you see on the command line interface or terminal is a file that can either be written to (changed) or read. Take a look at the image below:

This is a terminal showing the contents of the current user's directory (also known as a folder) using a common command ls. This is the user's "Home" folder and, even though these are more folders, the command line simply sees them as files. We can read these files, or write to / change these files (by adding more documents or changing their name, for example).

No matter what kind of file the folder holds, the command line will simply understand it as a file. This is a form of abstraction which allows the command line to generalize computation in a powerful way. This concept of command line abstraction is extremely important and you can read more about it in this e-book here.

Command Line Syntax

In this section, we'll be borrowing heavily from the Ubuntu Help website.

Please follow along with these examples in your own terminal!

The Prompt

The prompt shows that you've properly loaded your command line and it's ready to receive instructions. While it looks different on different computers, it is generally portrayed using the structure:

USERNAME@COMPUTER_NAME:~ $

So if you're name was Jaime and you were on a Ubuntu machine, your prompt would look like:

Jaime@ubuntu:~ $

Most important here is the dollar sign $ which represents the command line. In tutorials, you may see code that starts with the $. This means that you should copy everything after the $ but do not include the $. It is only meant to signify the code given should be executed on the command line.

Single Command

The most simple command possible on the terminal is the single command. To execute a single command on the terminal, simply type the command and press enter: 

$ COMMAND

We've already seen one example of this, the ls command, which lists all the files in the current directory. Here's another example:

$ pwd

This command lists the present working directory, or your current location in the computer, also known as your path. See the example below:

Another form of the single command is this format:

$ COMMAND

In this syntax, COMMAND operates on the ARGUMENT. See the examples below:

        user@ubuntu:~ $ echo "Hello World!"
        "Hello World!"
The command echo prints the argument "Hello World" to the command line output. 
        user@ubuntu:~ $ cat file1.txt
        These are the contents
        Of our test file
        Printed on the command line

The command cat prints the contents of the argument file1.txt to the command line output.

Here is another common format for command line code:

$ command -option
-option is a variable we pass to command to alter its behavior. For example, if we run: 
user@ubuntu:~ $ ls -a

This will show all the files, including those that are hidden. Compare this terminal screen to the previous ls example above or try it on your own terminal!

When option is used in conjunction with running a program from the command line, we call it a flag. A common one you'll see in the course is -v or --version, which tells us what version of the program we are running (note the difference in the number of dashes):

        user@ubuntu:~ $ node -v 
        v15.11.0

When we're troubleshooting issues, this is extremely helpful to know what version of a program we're running.

It's possible to run a command with five, ten or more flags. When we're working with Ethereum software clients, such as Geth or Hyperledger Besu, you will see they sometimes have many flags!

Multiple Commands

We can also run multiple commands in the same terminal line. We do this by the use of operators such as &&. The format for running multiple commands is:

$ command1 && command2

A common way this is used is tying together the mkdir (which makes a directory or folder) and cd (which navigates into a directory or folder) commands. See the example below:

user@ubuntu:~ $ mkdir test-folder && cd test-folder
order of the commands is extremely important -- we cannot run cd before mkdir in the example above.

Control Flow

One more command line syntax structure before we get to common commands, control flow. This is the ability to redirect the input or output of a command.

One example of this is the > operator, which redirects the output of a command as an argument to the second command. A common way that is used is the following:

        user@ubuntu:~ $ echo "I'm writing to a text file!"> file1.txt

Here, we're directing the output of the echo command (the text argument we passed in) into the file file1.txt. We don't have to create file1.txt beforehand, the command line will create it for us if it doesn't exist. Warning, if it does exist, it will overwrite existing content. Test to see you've successfully written to file1.txt by running:

        user@ubuntu:~ $ cat file1.txt
        I'm writing to a text file!

Another control flow operator which redirects content is the pipe or |, located typically on the backslash or \ key. It does a similar job to > operator but works on different commands. See the example from Ubuntu Help below:

user@ubuntu:~ $ ls | sort

This command lists all the files in the current directory but also sorts them by their first character. Try it on your command line to see how it works!
