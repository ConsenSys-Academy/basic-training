Javascript vs Python

However, we wanted to explain some of the similarities and differences you might find between the two languages as well as why we are emphasizing Javascript for the Bootcamp.

Similarities

Beyond some syntactic similarities, Javascript and Python have three things in common:

  • Lots of learning material for both languages. There are so many great, introductory courses available for free online for both Javascript and Python.
  • Eager Communities Both the Javascript and Python have eager, growing and vibrant communities that welcome beginners and host conferences and talks you can find online or perhaps nearby in-person.
  • Active Development Both Javascript and Python are dynamic languages, with new developments, optimizations and innovations occurring all the time. There will always be something new to learn!

Let's go over some of the differences the languages have.

Differences: Use Cases (sort of)

Python has a reputation for being used in more academic settings. For example, Python is typically used before Javascript for data processing, visualization and machine learning. However, Python is not just used for data crunching, there are plenty of folks who use Python for web develompent and, as we mentioned, a Pythonic language is used in some major DeFi applications on Ethereum.

Javascript has a reputation for being used more with web development, as we mentioned previously. This has to do with the speed of Javascript vs Python -- Python runs a bit slower, with more capacity for data visualization, while Javascript can be packaged to run very quickly in the browser.

Differences: Modules

A module, sometimes called a library, is a discrete piece of software developed to assist in a specific task that a language might not be able to do on its own. An analogy might be a browser extension's relationship to a browser: the extension needs to be installed by was developed by a third-party to extend the capacity of the browser

Modules can be extremely helpful, particularly when learning how to program. Python incorporates a wide variety of modules while Javascript has less modules available. (Node.js does provide modules for Javascript, but it's not provided natively and requires setup)

Differences: REPL

A REPL is an interactive computing environment, similar to a command line, that takes in user input and executes them against a language. REPLs are for specific languages, and they help troubleshoot or provide some realtime experience.

Python has a native REPL client that can be run from your command line once Python has been installed. Javascript does not have a native REPL client, but because all major web browsers run Javascript, you can find a REPL on your browser. Node.js also provides a REPL client, but, again, it's not native to Javascript and requires setup.

Uniformity

As we mentioned earlier, before ECMAScript 6, Javascript had some issues with uniformity. However, since then, it's been completely standardized and uniform.

Python also had a resolved issue with uniformity, which was the division of Python into Python 2 and Python 3. Python 3 was released in December 2008 and was backward-incompatible, meaning any software built with Python 3 could not run on Python 2 and vice versa. However, Python 2 was discontinued in 2020 and there is now only Python 3.

You may still see some older tutorials refer to Python 2 or Python 3. An easy, quick way to tell if a tutorial is written using Python 2 is the print statement, Python 3 will only accept print("string here"), while Python 2 can accept print with no parenthesis (print "string here").

Conclusion

With the explosion of web development and web frameworks, Javascript has become a powerful universal software language. However, some new learners may find it more intimidating and opinionated to deal with. It's not a stretch to say that Python is actually the more approachable and readable language and therefore is easier for learning. However, Javascript is the heart of the extremely powerful developments happening at the cutting edge of software.

Ultimately, the choice doesn't matter because if you become a software engineer or programmer, you'll have to know both. Whichever you choose will simply be the gateway to learning about computer programming generally, development environments, and how to build software.

The real question you should ask is: Which language do you feel you'll have a better chance of learning? Have you found a free online course you really like in Python? Does your friend know Javascript and will teach you? Is there a great project online you want to understand better written in Javascript or Python?
