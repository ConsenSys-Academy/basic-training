Introduction

Whereas the command line is great for one-off lines and quick workflow, the code editor is where people build larger programs and projects. We'll be focussing on GUI code editors, also called "visual" or "desktop" code editors. Visual code editors are much more approachable than the command line to many coming from a non-technical background.

At the most basic, a code editor is a place where you can write and edit all different kinds of code. However, today, code editors can be customized and enhanced to not only help you write code but also manage all the tasks around writing, maintaining and deploying code.

In fact, the most popular visual code editors today do so much more than write code, they may even be considered an Integrated Development Environment (IDE). With an editor like VSCode, for example, you can spin up a server, use Github, run a command line or debug a particular file. The capacity is quite impressive.

We'll be walking through Visual Studio Code (VSCode) in this module, since it's free, runs on Windows, Linux and Mac, and has a large number of extensions. However, there are so many great code editors out there, including Atom, Sublime Text, and IDEA (written originally for Java but now supporting many languages). If you're working with Python, a popular IDE which also functions as a visual editor, is PyCharm.
